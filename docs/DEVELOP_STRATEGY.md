# AiTrend 内容开发策略文档

## 开发目标
生成分布均衡、信息密集、叙述独特的高质量AI热点内容

## 数据源开发策略

### 1. 强制多源覆盖规则
```python
每批次必须满足：
- 至少来自 3 个不同数据源
- 单个数据源最多 40% 的内容
- 必须包含至少 1 个实时源（Twitter/PH）
- 必须包含至少 1 个深度源（HN/Reddit）
```

### 2. 数据源权重分配
| 数据源 | 占比 | 内容特点 |
|--------|------|----------|
| Product Hunt | 20% | 新产品、商业化方向 |
| HackerNews | 25% | 技术深度、开发者视角 |
| GitHub Trending | 20% | 开源项目、代码实现 |
| Reddit | 15% | 用户体验、实际反馈 |
| Tavily | 15% | 新闻资讯、行业动态 |
| Twitter | 5% | 实时热点、官方发布 |

### 3. 内容生成策略矩阵

**按项目类型分类开发：**

| 类型 | 关键词检测 | 内容角度 | 必含信息 |
|------|-----------|----------|----------|
| AI模型/工具 | model, llm, ai, gpt | 能力边界、使用成本、替代方案 | 参数规模、 pricing、对比评测 |
| 开发者工具 | dev, tool, cli, api | 集成方式、性能指标、学习曲线 | 安装命令、代码示例、benchmark |
| 产品/应用 | app, product, platform | 用户场景、商业模式、增长数据 | MAU、收入模式、目标用户 |
| 开源项目 | github, open source | 架构设计、社区活跃度、贡献指南 | star数、commit频率、roadmap |
| 学术研究 | paper, research, novel | 创新点、实验结果、实际意义 | 准确率提升、数据集、应用场景 |

### 4. 叙述风格开发

**禁止使用的表达方式：**
- ❌ "第一...第二...第三..."
- ❌ "从...来看"
- ❌ "针对痛点"
- ❌ "功能设计上"
- ❌ "试图在...之间找平衡"

**强制使用的具体表达：**
- ✅ 具体数字："处理速度提升50%"
- ✅ 具体场景："适合在通勤时随手刷"
- ✅ 具体命令："pip install xxx 即可"
- ✅ 具体反馈："HN用户提到在处理边界情况时..."

## 开发检查清单

每篇内容发布前必须通过：

- [ ] 来自不同数据源（非单一HN）
- [ ] 包含具体功能描述
- [ ] 包含使用场景说明
- [ ] 包含技术/实现细节
- [ ] 包含对比或数据
- [ ] 无空话套话（通过关键词检测）
- [ ] 叙述自然流畅（人工抽检）
