# AiTrend Skill v0.1.0 开发经验总结

> 开发时间：2026-02-01
> 开发者：屁屁虾🦞

---

## 📋 项目概述

AiTrend 是一个多源 AI 热点资讯收集器，为 AI 行业 KOL 提供普通人视角的实用内容。

**核心功能：**
- 多数据源挖掘（Twitter、Product Hunt、HackerNews、GitHub、Brave）
- AI 智能总结（Gemini 2.5 Flash）
- 亲民内容生成（口语化、场景化、实用主义）

---

## 🎯 核心经验

### 1. 内容挖掘：从"旧方法"到"新方法"

**❌ 旧方法的问题：**
- Brave 关键词搜索 → 只能找到已被媒体报道的"旧闻"
- GitHub Trending → 只能找到已有知名度的项目
- 结果：永远是那几个老面孔（cline、Copilot 等）

**✅ 新方法的优势：**
- **Twitter/X** → 发现"正在讨论但还没被报道"的内容
- **Product Hunt** → 发现"今天刚上线"的新产品
- **HackerNews** → 发现"开发者社区正在关注"的早期项目
- **结果**：真正的新内容，普通人还没听说过的工具

**关键洞察：**
> 用旧时代的关键词搜索方法，只能找到旧内容。要发现新东西，必须去"信息源头"。

---

### 2. 内容筛选：从"数据指标"到"AI 理解"

**❌ 只看数据指标的问题：**
- 按 star 数排序 → 永远是老项目
- 按时间排序 → 可能包含大量低质量项目
- 结果：KOL 每天发同样的内容

**✅ AI 理解创新点：**
- 让 Gemini 分析"这个项目解决了什么痛点"
- 让 Gemini 判断"有什么独特之处"
- 让 Gemini 评估"为什么值得关注"
- 结果：真正的信息挖掘，而非数据堆砌

**关键洞察：**
> AI 时代的挖掘方法：让 AI 理解内容价值，而不是只凭数据指标筛选。

---

### 3. 内容呈现：从"行业高度"到"亲民视角"

**❌ 行业高度的问题：**
- "Google Project EAT 基础设施计划" → 普通人看不懂
- "Starlink 卫星数据用于 AI 训练" → 对普通人无用
- "Anthropic IPO 估值 3500 亿" → 距离普通人太远

**✅ 亲民视角的要点：**
- **实用主义**："这对我有什么用"
- **生活化场景**：结合工作、学习、生活具体说明
- **立即可用**：现在就能体验，免费或低成本
- **口语化表达**：像朋友聊天，不用行业黑话

**关键洞察：**
> KOL 的受众是普通 AI 使用者，不是技术专家。内容要让粉丝觉得"马上就能用上"。

---

### 4. 格式优化：从"暴露方法"到"自然叙述"

**❌ 暴露方法的问题：**
```
一句话说清：XXX
对你有什么用：XXX
怎么用：XXX
```
→ 像说明书，没有情感，暴露"我们在用什么方法"

**✅ 自然叙述的要点：**
```
这周我发现了一个超酷的 XXX，它其实就是...
想象一下，你在工作的时候经常遇到 XXX 问题...
以前你得...，现在用这个工具，只要...
```
→ 像朋友聊天，有场景、有感受、有观点

**关键洞察：**
> 不要暴露"内容创作的方法底子"，要像发现好东西跟朋友分享。

## 🔧 技术经验

### 1. 纯标准库开发

**原则：** 零依赖，开箱即用

**替代方案：**
| 第三方库 | 标准库替代 |
|---------|-----------|
| aiohttp | http.client |
| beautifulsoup4 | 正则表达式 |
| pydantic | dataclasses |
| pyyaml | json |

**关键洞察：**
> OpenClaw Skill 必须使用纯标准库，用户可能没有 pip 权限。

---

### 2. 多数据源聚合

**数据源优先级：**
1. Twitter/X（最新 viral 内容）
2. Product Hunt（新产品发布）
3. HackerNews（开发者社区）
4. GitHub（开源项目）
5. Brave（全网热点）

**关键洞察：**
> 多源交叉验证，确保发现的内容有真实热度，不是单一平台的噪音。

---

### 3. 环境变量管理

**使用 .env 文件管理敏感信息：**
```bash
# API Keys
BRAVE_API_KEY=xxx
GEMINI_API_KEY=xxx
GITHUB_TOKEN=xxx
PRODUCTHUNT_TOKEN=xxx
TWITTER_AUTH_TOKEN=xxx
TWITTER_CT0=xxx
```

**关键洞察：**
> 敏感信息不要硬编码，使用环境变量 + .env 文件管理。

## 📊 关键决策记录

| 决策 | 原因 | 结果 |
|------|------|------|
| 使用多数据源 | 单一数据源只能找到旧内容 | 发现真正的新工具 |
| AI 理解创新点 | 只看数据指标找不到好东西 | 挖掘到有价值的新项目 |
| 亲民视角 | 受众是普通 AI 使用者 | 内容对粉丝有用 |
| 口语化叙述 | 避免暴露方法底子 | 像朋友聊天一样自然 |
| 纯标准库 | 用户可能没有 pip 权限 | 零依赖，开箱即用 |

## 📝 开发标准

### 必须遵守的原则

1. **零系统权限**
   - 不使用 pip 安装依赖
   - 只使用 Python 标准库

2. **内容质量**
   - 必须是普通人能用的工具
   - 必须解释"对你有什么用"
   - 必须包含链接地址

3. **格式规范**
   - 口语化连贯叙述
   - 像朋友聊天一样自然
   - 不暴露挖掘方法
